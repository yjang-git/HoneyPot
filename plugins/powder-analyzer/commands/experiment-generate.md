Orchestrate battery electrode powder characterization experiment workflow generation from user request in `$ARGUMENTS`.

## Configuration Options

- `ARGUMENTS`: 사용자 요청 원문. 대상 물질, 실험 목표, 제약 조건을 포함한다.
- `material` (required): 대상 물질 (예: NMC811, LFP, 흑연)
- `particle_size` (required): 입도 D50 (μm)
- `density` (required): 이론 밀도 (g/cm³)
- `goal` (optional): 실험 목표 (마찰계수, 유동성, 호퍼설계, DEM 등). 기본값: 전체 워크플로우
- `equipment_mode` (optional): **standard** (ASTM 규격 장비, 기본값) / **custom** (간이 장치)
- `standards` (optional): 적용할 ASTM 표준. 기본값: 자동 선정
- `context_path` (optional): 프로젝트 컨텍스트 경로. 기본값: `MyVault/projects/powder/`
- `resources_path` (optional): ASTM/논문 리소스 경로. 기본값: `MyVault/resources/powder/`

## Phase 1: 컨텍스트 파악

1. Parse `$ARGUMENTS` and extract experiment parameters.
   - 대상 물질, 입도, 밀도 확인 (필수)
   - 실험 목표 확인 (선택, 미지정 시 전체 5단계 워크플로우)
   - `equipment_mode` 확인 (미지정 시 standard)
   - 제약 조건 확인 (장비, 시간, 비용 등)
2. Read project context files.
   - `{context_path}/실험개요.md` 참조 (존재 시)
   - `{resources_path}/astm/` 및 `{resources_path}/paper/` 확인
3. Validate resources exist.
4. **(equipment_mode = custom인 경우)** 장비 사양 수집:
   - 사용자에게 각 장치의 실측 치수, 재질, 교정 상태 확인
   - `references/custom-apparatus.md` 체크리스트 기준으로 ASTM 대비 차이 기록
   - 허용 범위 초과 장치는 FLAG 후 계속 진행 (사유 기재)

## Phase 2: 실험 설계 (Executor)

1. Use Task tool with subagent_type="powder-analyzer::powder-analyzer"
   - Prompt: "Design powder characterization experiment workflow for {material} (D50={particle_size}μm, ρ={density}g/cm³). Goal: {goal}. Equipment mode: {equipment_mode}. Apply ASTM standards from {resources_path}/astm/. Reference papers from {resources_path}/paper/. Generate complete 5-step protocol. If custom mode, reference custom-apparatus.md for apparatus specs."
   - Expected output: 실험 프로토콜 문서 (5단계 워크플로우, ASTM 절차, DEM 캘리브레이션 계획)

## Phase 3: 검증 (Validator)

1. Use Task tool with subagent_type="powder-analyzer::powder-analyzer-validator"
   - Prompt: "Validate the following experiment protocol generated by Executor: {Phase 2 output}. Check: unit consistency, physical plausibility, ASTM compliance, mathematical accuracy, cross-consistency, literature comparison. Reference ASTM files in {resources_path}/astm/ and papers in {resources_path}/paper/."
   - Expected output: 검증 보고서 (PASS/REVIEW/REJECT, 6개 카테고리별 판정)

## Phase 4: 독립 재현 (Replicator)

1. Use Task tool with subagent_type="powder-analyzer::powder-analyzer-replicator"
   - Prompt: "Independently design powder characterization experiment for {material} (D50={particle_size}μm, ρ={density}g/cm³). DO NOT reference the Executor's output. Use only ASTM standards from {resources_path}/astm/ and papers from {resources_path}/paper/. After completing independent design, compare with Executor result: {Phase 2 output}."
   - Expected output: 독립 결과 + 비교 보고서 (Match Ratio)

## Phase 5: Consensus Gate (합의 관문)

1. Evaluate 3-AI consensus:
   - Validator verdict == PASS or REVIEW?
   - Replicator Match Ratio >= 80%?
2. Decision:
   - **PASS**: Validator PASS + Match Ratio ≥ 80% → 최종 프로토콜 확정
   - **CONDITIONAL PASS**: Validator REVIEW + Match Ratio ≥ 95% → FLAG 항목 주석 추가 후 확정
   - **FAIL**: Validator REJECT 또는 Match Ratio < 80% → 불일치 원인 분석 후 Phase 2부터 재실행 (최대 2회)

## Phase 6: 결과 전달

1. Generate final experiment protocol document.
   - 검증 완료된 5단계 실험 프로토콜
   - 검증 보고서 요약 (Validator + Replicator)
   - Consensus Gate 판정 결과
2. Save output.
   - 프로토콜: `{context_path}/실험프로토콜_{material}.md`
   - 검증보고서: `{context_path}/검증보고서_{material}.md`
3. Provide next actions.
   - 실험 실행 가이드
   - 데이터 수집 후 DEM 캘리브레이션 방법

## MUST DO

- [ ] `$ARGUMENTS`에서 물질 정보와 실험 목표를 정확히 파악한다.
- [ ] 3-AI 시스템 (Executor → Validator → Replicator → Consensus)을 반드시 순서대로 실행한다.
- [ ] Consensus Gate 판정 결과를 사용자에게 명확히 보고한다.
- [ ] 모든 출력 문서에 ASTM 표준 번호와 논문 출처를 명시한다.

## MUST NOT DO

- [ ] Consensus Gate를 건너뛰고 Executor 결과만 전달하지 않는다.
- [ ] Replicator에게 Executor 출력을 사전에 제공하지 않는다.
- [ ] 검증 실패 시 무시하고 진행하지 않는다.
- [ ] ASTM 원문 없이 표준 절차를 추측하여 작성하지 않는다.

## Usage Example

```
@powder-analyzer NMC811 파우더 물성 분석 실험 설계해줘.

ARGUMENTS:
- material: NMC811
- particle_size: 5
- density: 4.78
- goal: 마찰계수 측정 + DEM 캘리브레이션
```
