# Seminar 스타일 테마 팔레트

세미나/발표 자료를 위한 9가지 무드 테마. 각 테마는 학술 발표와 전문 세미나에 적합한 4색 팔레트를 제공합니다.

**시각 정체성**: "Nature 저널 Figure처럼 — 패널 라벨이 붙은 학술 다이어그램"

**색상 전략**: 각 무드별 4색 팔레트를 사용하되, 주조색에서 보조색으로의 그라데이션 변화를 허용합니다. concept 스타일의 단색 접근과 차별화되는 부드러운 색상 전환이 특징입니다.

## 렌더링 스타일 (Rendering Style)

이 스타일의 시각적 DNA를 정의합니다. prompt-designer가 INSTRUCTION/CONFIGURATION 블록 생성 시 반드시 참조해야 합니다.

| 차원 | 지시 사항 |
|------|-----------|
| **서피스 (박스/패널)** | 1px 테두리 + 연한 배경색 채움 + 소프트 드롭섀도우(4px blur). 카드 느낌. 각 카드 좌상단에 (a), (b), (c) 패널 라벨을 부착한다. |
| **배경** | 미세한 도트 그리드 패턴 배경(5% opacity 도트). 학술 논문의 Figure 배경처럼 미묘한 그리드감을 준다. 좌하단에 작은 "Figure N." 캡션 라벨. |
| **코너/엣지** | 소프트 라운딩(8px radius). 모든 카드, 박스에 일관된 8px 라운딩을 적용한다. 부드럽지만 과하지 않은 학술적 정돈감. |
| **연결선** | 1.5px 실선 + 열린 삼각형 화살표(▷). 학술 다이어그램 표준. 연결선 색상은 보조색을 사용한다. 곡선(Bezier) 연결을 허용한다. |
| **시각 장식** | 라인 아이콘(outline 스타일, 1.5px stroke). 학술 차트 요소(축, 범례, 데이터 포인트). 각주 번호. 패널 라벨 (a) (b). 색상 범례 박스. |
| **타이포그래피** | 4단 위계(제목/소제목/본문/캡션). 제목은 볼드, 소제목은 세미볼드+보조색, 본문은 레귤러, 캡션/각주는 이탤릭+작은 크기. 학술 논문 Figure caption 스타일. |
| **공간 구성** | 비대칭 2컬럼(70:30 또는 60:40). 메인 다이어그램 70% + 사이드 범례/설명 30%. 또는 상단 다이어그램 60% + 하단 데이터 테이블 40%. 학술 Figure 배치. |
| **시각 메타포** | 학술 다이어그램(Scientific Diagram). 노드-엣지 그래프, 플로우차트, 데이터 시각화(scatter, bar, line). 축+라벨+범례 세트. Figure panel 배치. Nature/Science 저널 Figure 스타일. |

## 권장 레이아웃

seminar 스타일에 특히 적합한 레이아웃 우선순위:

| 우선순위 | 레이아웃 | 이유 |
|:--------:|----------|------|
| 1 | Flow (플로우) | 연구 방법론, 데이터 파이프라인 — 학술 발표의 핵심 도식 |
| 2 | Structure (구조) | 시스템 아키텍처, 모델 구조 — 기술 발표에 빈번 |
| 3 | Card-Grid (카드 그리드) | 실험 결과, 사례 비교 — 패널 (a)(b)(c) 배치에 최적 |
| 4 | Strategy Map (전략맵) | 연구 포지셔닝, 기술 비교 매트릭스 — 학술적 분석 도구 |

**비권장**: Single-Hero, Z-Pattern (학술 발표에 비해 정보량 과소)

## 콘텐츠 표현 규칙

seminar 스타일의 텍스트는 **학술 발표체**를 따릅니다:

| 규칙 | 설명 | 예시 |
|------|------|------|
| ~다 체 | 학술 발표의 서술형 종결 | ✗ "비용 절감" → ✓ "비용이 30% 절감된다" |
| 완전한 문장 | 불릿 포인트마다 1개의 완전한 문장 | ✗ "AI 품질관리" → ✓ "AI 기반 품질관리 시스템을 제안한다" |
| 근거-결론 쌍 | 주장과 근거를 쌍으로 배치 | "정확도 99.2% → 기존 대비 15%p 향상" |
| 출처/각주 | 데이터에 각주 번호 사용 권장 | "시장 규모 5조원¹" |
| 전문용어+풀이 | 전문용어를 쓰되 간단한 풀이 병행 | "CNN(합성곱 신경망) 기반 결함 탐지" |

## 렌더링 금지 요소 (CRITICAL)

프롬프트 생성 시 다음 요소는 **CONTENT BLOCK에 절대 포함 금지**:

| 유형 | 금지 예시 | 설명 |
|------|-----------|------|
| 위치 지시자 | `[상단]`, `[하단 결론1]`, `[왼쪽 영역]` | 레이아웃 배치 힌트 |
| 레이아웃 유형명 | `세미나 프레젠테이션`, `강연 슬라이드` | 메타포/레이아웃 이름 |
| 색상 코드 | `(#HEXCODE)`, `Primary Color` | CONFIGURATION에서만 사용 |
| 크기 힌트 | `(대형)`, `(중형)`, `48pt` | INSTRUCTION에서만 사용 |
| 역할 라벨 | `Main Title`, `Sub-header` | INSTRUCTION에서만 사용 |

CONTENT에는 **순수한 텍스트 내용만** 포함합니다.
