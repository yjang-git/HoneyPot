# Pitch (피치덱) 테마

**용도**: 간결하고 임팩트 있는 피치덱 스타일의 시각자료를 생성합니다. 문제→해결→시장→팀 흐름으로 투자자/의사결정자를 설득합니다.

**시각 정체성**: "Stripe 랜딩페이지처럼 — 거대한 숫자와 보라빛 그래디언트"

**활용 상황**:
- 스타트업 투자 유치 발표
- 신규 프로젝트 제안서
- 경영진 대상 1-pager
- 데모데이/IR 발표 자료
- 핵심 성과 요약 슬라이드

---

## 색상 팔레트

| 역할 | 색상 | HEX | 용도 |
|------|------|-----|------|
| 주조 | 딥 네이비 | #0A2540 | 제목, 핵심 메시지, 숫자 강조 |
| 보조 | 쿨 그레이 | #425466 | 본문, 보조 텍스트, 구분선 |
| 강조 | 일렉트릭 블루 | #635BFF | CTA, 핵심 수치, 하이라이트 |
| 배경 | 퓨어 화이트 | #FFFFFF | 전체 배경, 여백 강조 |

**팔레트 특징**:
- Stripe/현대 핀테크 스타일의 세련된 조합
- 고대비로 핵심 숫자와 메시지가 즉각 인지됨
- 블루 강조색이 신뢰감 + 혁신 이미지 동시 전달

## 권장 레이아웃

### Z-Pattern (최우선 권장)
- **(1) 좌상**: 핵심 문제 또는 제목
- **(2) 우상**: 임팩트 숫자 (시장 규모, 성장률 등)
- **(3) 좌하**: 솔루션 핵심 (한 문장)
- **(4) 우하**: CTA 또는 요청사항

```
(1) [문제/제목] ─────────────── (2) $10B+ 시장
        \
         \
(3) [솔루션 한 줄] ─────────────── (4) [CTA/요청]
```

**장점**: 30초 내 핵심 메시지 전달, 시선 흐름 최적화

### Single-Hero (임팩트 최대화)
- 화면 중앙에 **하나의 대형 숫자/메시지**만 배치
- 하단에 한 줄 부연 설명

```
┌─────────────────────────────┐
│                             │
│          +340%              │
│      월간 활성 사용자 증가     │
│                             │
└─────────────────────────────┘
```

## 스타일 노트

### 숫자 강조 규칙
- **크기**: 핵심 숫자는 화면의 30% 이상 차지
- **단위**: 숫자 옆에 단위 명시 ($, %, 배, 명 등)
- **맥락**: 숫자 아래 한 줄로 의미 설명

### 텍스트 원칙
- **한 슬라이드 한 메시지**: 복수 아이디어 배제
- **동사 사용**: "AI 기반 솔루션" → "AI로 해결합니다"
- **구체적 수치**: "크게 성장" → "3배 성장"

### 여백 활용
- **최소 30% 여백**: 여백이 메시지를 강조
- **밀집 배제**: 빽빽한 내용은 피치덱에 부적합

## 피치덱 슬라이드 시퀀스 (참고)

| 순번 | 슬라이드 | 핵심 요소 |
|:----:|----------|-----------|
| 1 | 문제 | 고객의 Pain Point 한 문장 |
| 2 | 솔루션 | 우리 해결책 한 문장 |
| 3 | 시장 | TAM/SAM/SOM 또는 시장 규모 |
| 4 | 비즈니스 모델 | 수익 구조 간략화 |
| 5 | 트랙션 | 핵심 지표 1-3개 |
| 6 | 팀 | 핵심 인력 소개 |
| 7 | Ask | 투자 요청액 + 사용처 |

## 적합/부적합 케이스

**적합**: 투자자/VC 대상 발표, 경영진 의사결정 요청, 핵심 성과 1-pager, 데모데이/IR 자료, 프로젝트 킥오프 요약

**부적합**: 상세 기술 아키텍처 설명, 10개 이상 항목 나열, 복잡한 프로세스 플로우, 학술/연구 보고서, 비교 분석 → `theme-comparison`

## 렌더링 스타일 (Rendering Style)

이 스타일의 시각적 DNA를 정의합니다. prompt-designer가 INSTRUCTION/CONFIGURATION 블록 생성 시 반드시 참조해야 합니다.

| 차원 | 지시 사항 |
|------|-----------|
| **서피스 (박스/패널)** | 테두리 없는 라운드 카드 + 강한 드롭섀도우(8px blur, 20% opacity). Stripe UI처럼 카드가 배경에서 살짝 떠 있는 느낌. 배경과 카드의 명확한 레이어 분리. |
| **배경** | 순백 배경에 좌상단 코너에 보라빛(#635BFF → 투명) 그래디언트 블롭 장식. 또는 딥 네이비(#0A2540) → 일렉트릭 블루(#635BFF) 미묘한 방사형 그래디언트. |
| **코너/엣지** | 중간 라운딩(12px radius). Stripe 카드 스타일. 숫자 강조 블록은 pill shape(완전 라운드) 허용. |
| **연결선** | 연결선 최소화. 쓸 경우 1px 쿨그레이(#425466) 점선. 숫자와 메시지가 자체로 연결을 나타낸다. 복잡한 다이어그램 자체를 지양한다. |
| **시각 장식** | 그래디언트 블롭(유기적 도형) 배경 장식 1-2개. 핵심 숫자 뒤에 큰 반투명 원형 하이라이트. 미니 스파크라인 차트(↗). |
| **타이포그래피** | 숫자가 가장 크다. 핵심 메트릭(+340%, $10B)이 화면의 30% 이상 차지. 제목보다 숫자가 크다. 제목은 세미볼드 중간 크기. CTA는 강조색(#635BFF) 볼드. |
| **공간 구성** | 대담한 비대칭. 핵심 숫자가 좌측 또는 중앙 상단에 매우 크게. 나머지 요소들이 빈 공간에 작게 흩어진다. Z-패턴 시선 유도. 30% 이상 의도적 빈 공간. |
| **시각 메타포** | 모던 대시보드(Modern Dashboard). 큰 숫자 카드, 미니 스파크라인, 진행률 바. SaaS 제품 스크린샷 느낌. Dribbble/Behance의 UI 목업 스타일. |

## 렌더링 금지 요소 (CRITICAL)

프롬프트 생성 시 다음 요소는 **CONTENT BLOCK에 절대 포함 금지**:

| 유형 | 금지 예시 | 설명 |
|------|-----------|------|
| 위치 지시자 | `[상단]`, `[하단 결론1]`, `[왼쪽 영역]` | 레이아웃 배치 힌트 |
| 레이아웃 유형명 | `Z-Pattern`, `Pitch Deck Layout` | 메타포/레이아웃 이름 |
| 색상 코드 | `(#HEXCODE)`, `Primary Color` | CONFIGURATION에서만 사용 |
| 크기 힌트 | `(대형)`, `(중형)`, `48pt` | INSTRUCTION에서만 사용 |
| 역할 라벨 | `Main Title`, `Sub-header` | INSTRUCTION에서만 사용 |

CONTENT에는 **순수한 텍스트 내용만** 포함합니다.
