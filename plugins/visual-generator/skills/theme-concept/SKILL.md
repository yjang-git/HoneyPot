# Concept 스타일 테마 팔레트

TED 스타일 미니멀 인포그래픽을 위한 9가지 무드 테마. 각 테마는 특정 개념 유형과 맥락에 최적화된 4색 팔레트를 제공합니다.

**시각 정체성**: "Apple Keynote처럼 — 칠흑 같은 여백 속 하나의 메시지가 떠 있다"

**색상 전략**: 모노크롬 + 단일 강조색. 주조색 계열의 단색 톤으로 통일하고, 강조색 1가지만 포인트로 사용합니다. 그라데이션은 사용하지 않습니다.

## 렌더링 스타일 (Rendering Style)

이 스타일의 시각적 DNA를 정의합니다. prompt-designer가 INSTRUCTION/CONFIGURATION 블록 생성 시 반드시 참조해야 합니다.

| 차원 | 지시 사항 |
|------|-----------|
| **서피스 (박스/패널)** | 테두리 없는 플로팅 텍스트. 박스를 쓰지 않는다. 텍스트와 도형만으로 구성. 구분이 필요하면 여백으로만 구분한다. |
| **배경** | 순백(#FFFFFF) 또는 순흑(#111111) 단색 배경만 허용. 배경에 어떤 장식(패턴, 그라데이션, 도형)도 넣지 않는다. |
| **코너/엣지** | 박스를 사용하지 않으므로 코너 자체가 없다. 도형을 쓸 경우 완전한 원(circle) 또는 직선(line)만 허용. 중간 라운딩(8px, 12px 등) 사용하지 않는다. |
| **연결선** | 초가는 1px 실선만 허용. 화살표 사용 최소화. 쓸 경우 삼각형 없는 얇은 직선만. 가능하면 연결선 대신 근접 배치로 관계를 암시한다. |
| **시각 장식** | 장식 제로(zero decoration). 아이콘 없음. 일러스트 없음. 텍스트와 기하학적 도형(원, 선, 직사각형)만 사용. 타이포그래피의 크기 대비가 장식 역할을 대신한다. |
| **타이포그래피** | 극단적 2단 크기 대비. 제목은 화면의 40-50%를 차지하는 초대형. 보조 텍스트는 제목의 1/6 크기. 2가지 크기(대/소)만 사용하고 중간 크기는 쓰지 않는다. 제목 볼드, 나머지 레귤러. |
| **공간 구성** | 중앙 집중 + 40% 이상 여백. 콘텐츠가 화면 중앙 60% 영역에 집중된다. 상하좌우 넓은 여백. 요소 간 간격이 넓다. 숨 쉴 공간이 디자인의 핵심이다. **대칭 규칙**: 좌우 배치 시 양쪽 항목 수·크기·간격을 정확히 동일하게 맞춘다. 상하 배치 시에도 동일 원칙 적용. |
| **시각 메타포** | 추상 기하학(Abstract Geometric). 원, 직선, 직사각형만으로 개념을 표현. 벤 다이어그램, 동심원, 직선 플로우. 일체의 사실적 요소를 배제하고 개념의 순수한 구조만 보여준다. **완결성 규칙**: Contrast 레이아웃 사용 시 좌우 영역의 시각적 무게(요소 수, 도형 크기, 색상 면적)를 균형있게 유지한다. 모든 도형과 텍스트는 정렬 그리드에 맞춰 배치한다. |

---

### 대칭 및 시각 완결성 규칙

concept 스타일의 미니멀리즘은 **불균형이 즉각 드러나므로** 대칭과 완결성이 특히 중요합니다:

- **좌우 대칭**: 좌우 분할 배치 시 양쪽 항목 수, 도형 크기, 텍스트 길이, 간격을 정확히 동일하게 맞춘다
- **상하 대칭**: 상하 분할 시에도 동일 원칙. 상단과 하단의 시각적 무게가 균등해야 한다
- **요소 수 매칭**: 한쪽에 3개 항목이면 반대쪽도 반드시 3개. 비대칭 항목 수 금지
- **시각적 무게 균형**: Contrast 레이아웃에서 좌우 영역의 도형 크기, 색상 면적, 텍스트 분량이 균형을 이룬다
- **정렬 일관성**: 모든 요소는 보이지 않는 그리드 위에 정렬한다. 임의 배치 금지
- **여백 균등**: 상하좌우 여백이 동일해야 한다. 한쪽만 넓은 여백은 불균형을 유발한다

## 권장 레이아웃

concept 스타일에 특히 적합한 레이아웃 우선순위:

| 우선순위 | 레이아웃 | 이유 |
|:--------:|----------|------|
| 1 | Central (중심) | 하나의 핵심을 둘러싼 단순한 구조 — concept의 미니멀리즘에 최적 |
| 2 | Section-Flow (섹션-플로우) | 상단 대형 메시지 + 하단 핵심 흐름 — 여백과 메시지 집중 가능 |
| 3 | Bento Grid (벤토 그리드) | Hero 타일 1개 + 보조 타일 — 미니멀한 정보 배치 |
| 4 | Concentric (동심원) | 추상 기하학 원형 — concept의 도형 중심 메타포에 적합 |

**비권장**: Swimlane, Org-Network, Horizontal Timeline (정보 밀도 과다)

## 콘텐츠 표현 규칙

concept 스타일의 텍스트는 **극도로 절제**됩니다:

| 규칙 | 설명 | 예시 |
|------|------|------|
| 키워드 중심 | 완전한 문장 금지. 명사 또는 명사구만 사용 | ✗ "AI가 품질을 관리합니다" → ✓ "AI 품질관리" |
| 3단어 이내 | 하나의 텍스트 요소는 3단어를 넘지 않음 | ✗ "실시간 데이터 기반 품질 예측" → ✓ "실시간 품질예측" |
| 숫자는 단독 | 숫자는 설명 없이 크게 배치. 맥락은 위치로 전달 | ✗ "정확도 99.2% 달성" → ✓ "99.2%" |
| 서술어 배제 | ~합니다, ~입니다, ~한다 등 종결어미 사용 금지 | ✗ "비용을 절감합니다" → ✓ "비용 절감" |

## 렌더링 금지 요소 (CRITICAL)

프롬프트 생성 시 다음 요소는 **CONTENT BLOCK에 절대 포함 금지**:

| 유형 | 금지 예시 | 설명 |
|------|-----------|------|
| 위치 지시자 | `[상단]`, `[하단 결론1]`, `[왼쪽 영역]` | 레이아웃 배치 힌트 |
| 레이아웃 유형명 | `TED 스타일`, `미니멀 인포그래픽` | 메타포/레이아웃 이름 |
| 색상 코드 | `(#HEXCODE)`, `Primary Color` | CONFIGURATION에서만 사용 |
| 크기 힌트 | `(대형)`, `(중형)`, `48pt` | INSTRUCTION에서만 사용 |
| 역할 라벨 | `Main Title`, `Sub-header` | INSTRUCTION에서만 사용 |

CONTENT에는 **순수한 텍스트 내용만** 포함합니다.
